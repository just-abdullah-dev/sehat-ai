1. Project Overview
The Sehat-AI platform detects two diseases — Tuberculosis (TB) and Pneumonia — from chest X-ray images using separate deep learning models. A critical problem was identified: when users upload irrelevant images (selfies, documents, brain MRIs, etc.), the disease detection models still produce a prediction instead of rejecting the input.

This document covers the training pipeline for a binary gatekeeper model that sits in front of both disease models. Its sole job is to answer one question:

“Is this image a chest X-ray or not?”

Why a separate validator model?
Disease detection models are trained on medical images only. They have never seen cats, buildings, or random photos, so they have no concept of "invalid input". They will always output a score regardless. A dedicated binary classifier, trained on both X-rays and unrelated images, is the only reliable way to gate irrelevant uploads.

1.1 System Architecture
The validator is the first layer in the inference pipeline. Only images that pass the validator are forwarded to the disease models.

Stage	Description
1. Upload	User uploads any image through the frontend
2. Validator	chest_xray_validator.h5 runs inference — returns 0.0 to 1.0
3. Gate check	Score compared against saved threshold (from metadata JSON)
4a. Reject	Score below threshold — return 400 error with clear message
4b. Route	Score above threshold — forward to TB or Pneumonia model
5. Result	Disease prediction returned to the user

2. Dataset Strategy
The model is trained on two classes. Choosing the right data for each class is critical to building a validator that rejects both completely unrelated images AND medical images that are not chest X-rays.

2.1 Class 1 — Positive (chest_xray/)
Source: Paul Mooney chest-xray-pneumonia dataset on Kaggle. This is the same dataset already used for the Pneumonia detection model, so no new download is needed for positives. Both NORMAL and PNEUMONIA subfolders are used since the validator only cares whether the image is a chest X-ray, not about disease status.

•	Train/NORMAL — healthy chest X-rays
•	Train/PNEUMONIA — pneumonia chest X-rays
•	Test/NORMAL and Test/PNEUMONIA — for additional variety

2,000 images are sampled from this pool to keep the dataset balanced and training fast on Colab free tier.

2.2 Class 2 — Negative (not_xray/)
The negative class must cover two distinct types of non-chest-X-ray images:

Type A — Completely unrelated images: Natural scene photos (buildings, forests, streets, mountains). Source: Intel Image Classification dataset on Kaggle (~1,500 images sampled).

Type B — Medical but not chest X-rays: Brain MRI scans. These are grayscale and look medical, so disease models would likely misclassify them. Source: Brain MRI Images for Brain Tumor Detection dataset on Kaggle (~500 images sampled).

Why include brain MRIs in negatives?
If the negative class only contains natural photos, the model learns to reject "colorful outdoor images" rather than learning what specifically makes something a chest X-ray. Brain MRIs are grayscale and structured like medical scans — adding them forces the model to learn chest-specific anatomy, not just "grayscale vs color".

2.3 Final Dataset Balance
Class	Count
chest_xray/ (positive)	~2,000 images
not_xray/ (negative)	~2,000 images (1,500 Intel + 500 Brain MRI)
Total	~4,000 images
Train / Validation split	80% / 20% (via ImageDataGenerator)

Augmentation explained
Parameter	Purpose
rescale=1./255	Normalizes pixel values from [0,255] to [0,1]. Neural networks train better on small values
rotation_range=10	Rotates image up to 10 degrees — handles slightly tilted X-rays
brightness_range=[0.85, 1.15]	Simulates over/under-exposed X-rays from different machines
zoom_range=0.1	Random 10% zoom — handles different patient distances from detector
horizontal_flip=True	Flips image left-right — both orientations of chest X-rays are valid
fill_mode='nearest'	Fills empty pixels after rotation/shift using edge pixel values

Important: note the class_indices output
The output shows something like {"chest_xray": 0, "not_xray": 1}. This tells you which integer label maps to which class. The order is alphabetical — chest_xray comes before not_xray so it gets label 0. Save this information — you will need it in your backend to interpret model output correctly.

Cell 5 — Build Model
What this cell does
Constructs the neural network using Transfer Learning. MobileNetV2, pre-trained on ImageNet (1.4 million images, 1,000 classes), is used as the feature extractor. A custom classification head is added on top for binary output.

Code
def build_model():
    base = MobileNetV2(
        input_shape=(224, 224, 3),
        include_top=False,
        weights='imagenet'
    )
    base.trainable = False  # freeze base initially
 
    model = models.Sequential([
        base,
        layers.GlobalAveragePooling2D(),
        layers.BatchNormalization(),
        layers.Dense(128, activation='relu'),
        layers.Dropout(0.4),
        layers.Dense(64, activation='relu'),
        layers.Dropout(0.2),
        layers.Dense(1, activation='sigmoid')
    ])
 
    model.compile(
        optimizer=tf.keras.optimizers.Adam(learning_rate=1e-3),
        loss='binary_crossentropy',
        metrics=['accuracy',
                 tf.keras.metrics.AUC(name='auc'),
                 tf.keras.metrics.Precision(name='precision'),
                 tf.keras.metrics.Recall(name='recall')]
    )
    return model
 
model = build_model()
model.summary()

Architecture explained
Layer	Purpose
MobileNetV2 (frozen)	Extracts visual features — edges, textures, shapes. 2.3M parameters already trained on ImageNet
GlobalAveragePooling2D	Converts the 7x7x1280 feature map to a flat 1280-vector by averaging each channel
BatchNormalization	Normalizes activations to stabilize and speed up training
Dense(128, relu)	Learns which combinations of features indicate a chest X-ray
Dropout(0.4)	Randomly deactivates 40% of neurons during training to prevent overfitting
Dense(64, relu)	Second feature combination layer for finer discrimination
Dropout(0.2)	Lighter regularization on the smaller layer
Dense(1, sigmoid)	Output neuron — sigmoid squashes output to [0,1], interpreted as probability

Why MobileNetV2?
•	Lightweight — 3.4MB model size, fast inference on CPU in your backend
•	Strong feature extraction — trained on 1.4M diverse images
•	include_top=False removes the ImageNet classification head, keeping only the feature extractor
•	Works well for medical imaging tasks with limited training data

Cell 6 — Phase 1 Training (Frozen Base)
What this cell does
Trains only the custom classification head while keeping MobileNetV2's weights frozen. This is the standard first phase of transfer learning — the head adapts to the new task without destabilizing the pre-trained features.

Code

Callbacks explained
Callback	Behaviour
EarlyStopping	Stops training if val_auc does not improve for 4 consecutive epochs. Restores the best weights automatically — no manual checkpoint loading needed
ReduceLROnPlateau	Halves the learning rate if val_loss plateaus for 2 epochs. Helps the model find a better minimum without overshooting
ModelCheckpoint	Saves weights to best_model.h5 every time val_auc improves. Acts as a safety save in case of Colab disconnection

Cell 7 — Phase 2 Fine-tuning
What this cell does
Unfreezes the last 30 layers of MobileNetV2 and trains them together with the classification head at a very low learning rate. This allows the feature extractor to adapt its higher-level features to medical image characteristics.

Code
model.load_weights('/content/best_model.h5')
 
# Unfreeze the last 30 layers of MobileNetV2
base_model = model.layers[0]
base_model.trainable = True
for layer in base_model.layers[:-30]:
    layer.trainable = False
 
model.compile(
    optimizer=tf.keras.optimizers.Adam(learning_rate=1e-5),
    loss='binary_crossentropy',
    metrics=['accuracy', tf.keras.metrics.AUC(name='auc'),
             tf.keras.metrics.Precision(name='precision'),
             tf.keras.metrics.Recall(name='recall')]
)
 
callbacks_phase2 = [
    EarlyStopping(
        monitor='val_auc', patience=5,
        restore_best_weights=True, mode='max'
    ),
    ReduceLROnPlateau(
        monitor='val_loss', factor=0.3,
        patience=2, min_lr=1e-7, verbose=1
    ),
    ModelCheckpoint(
        '/content/best_model.h5', monitor='val_auc',
        save_best_only=True, mode='max', verbose=1
    )
]
 
print('Phase 2: Fine-tuning last 30 layers...')
history2 = model.fit(
    train_gen,
    validation_data=val_gen,
    epochs=10,
    callbacks=callbacks_phase2
)

Why 1e-5 learning rate?
Phase 1 uses lr=1e-3 because the head starts random and needs large updates to converge. Phase 2 uses lr=1e-5 (100x smaller) because MobileNetV2's weights are already well-trained — large updates would destroy the learned features. This technique is called discriminative fine-tuning.

Why unfreeze only the last 30 layers?
MobileNetV2 has 154 layers. Early layers detect generic patterns (edges, colors, gradients) that are universal across all image types. Later layers detect high-level, task-specific patterns. By unfreezing only the last 30 layers, we adapt the high-level features to medical imagery while preserving the generic low-level features.

What to look for in the plots
•	Accuracy / AUC: training and validation curves should converge. If training >> validation, the model is overfitting
•	Loss: both curves should decrease and level off. A rising validation loss with decreasing training loss is a classic overfitting signal
•	Fine-tune start line: you should see a small jump in performance after the fine-tuning phase begins
•	Target: val_accuracy > 0.95, val_auc > 0.98 for this binary classification task

Cell 9 — Confusion Matrix & Threshold Selection
What this cell does
Finds the optimal classification threshold by testing values from 0.3 to 0.9 and selecting the one that maximises F1-score. Plots a confusion matrix and prints a full classification report.

Code
from sklearn.metrics import (confusion_matrix, classification_report,
                              f1_score, precision_score,
                              recall_score, accuracy_score)
import seaborn as sns
 
val_gen.reset()
y_pred_probs = model.predict(val_gen, verbose=1).flatten()
y_true = val_gen.classes
 
thresholds = np.arange(0.3, 0.9, 0.05)
print('Threshold | Accuracy | Precision | Recall | F1')
print('-' * 55)
best_threshold, best_f1 = 0.5, 0
 
for t in thresholds:
    y_pred = (y_pred_probs >= t).astype(int)
    acc   = accuracy_score(y_true, y_pred)
    prec  = precision_score(y_true, y_pred, zero_division=0)
    rec   = recall_score(y_true, y_pred, zero_division=0)
    f1    = f1_score(y_true, y_pred, zero_division=0)
    print(f'  {t:.2f}    |  {acc:.3f}   |   {prec:.3f}   |  {rec:.3f} | {f1:.3f}')
    if f1 > best_f1:
        best_f1, best_threshold = f1, t
 
print(f'\nBest threshold: {best_threshold:.2f} (F1={best_f1:.4f})')
 
y_pred_best = (y_pred_probs >= best_threshold).astype(int)
cm = confusion_matrix(y_true, y_pred_best)
class_names = list(val_gen.class_indices.keys())
 
plt.figure(figsize=(6, 5))
sns.heatmap(cm, annot=True, fmt='d', cmap='Blues',
            xticklabels=class_names, yticklabels=class_names)
plt.title(f'Confusion Matrix (threshold={best_threshold:.2f})')
plt.ylabel('True label')
plt.xlabel('Predicted label')
plt.tight_layout()
plt.savefig('/content/confusion_matrix.png', dpi=150)
plt.show()
 
print(classification_report(y_true, y_pred_best,
                            target_names=class_names))

Understanding the threshold
The model outputs a raw probability between 0 and 1. A threshold converts it to a binary prediction. The default 0.5 is rarely optimal. For this medical validator:

•	A false negative (non-X-ray passes through as X-ray) is more harmful — it sends garbage to a disease model and produces a fake diagnosis
•	A false positive (valid X-ray gets rejected) is less harmful — the user just needs to re-upload
•	For this reason, consider manually increasing the best_threshold by 0.05 to 0.10 above the F1-optimal value for extra safety in production

4.1 Model Loading & Inference
import json
import numpy as np
from PIL import Image
import io
from tensorflow.keras.models import load_model
 
# Load metadata
with open('models/chest_xray_validator_metadata.json') as f:
    meta = json.load(f)
 
THRESHOLD = meta['best_threshold']
CLASS_INDICES = meta['class_indices']
# e.g. {'chest_xray': 0, 'not_xray': 1}
# Score >= THRESHOLD means class index 1 (not_xray) if not_xray=1
# Always derive XRAY_CLASS from metadata, never hardcode
XRAY_CLASS_INDEX = CLASS_INDICES['chest_xray']
 
# Load all models once at startup
validator = load_model('models/chest_xray_validator.h5')
tb_model  = load_model('models/tb_model.h5')
pn_model  = load_model('models/pneumonia_model.h5')
 
def preprocess(image_bytes, size=(224, 224)):
    img = Image.open(io.BytesIO(image_bytes)).convert('RGB')
    img = img.resize(size)
    arr = np.array(img) / 255.0
    return np.expand_dims(arr, axis=0)
 
def validate_xray(image_bytes):
    arr = preprocess(image_bytes)
    score = float(validator.predict(arr, verbose=0)[0][0])
    # If chest_xray=0, score close to 0 means chest X-ray
    # Adjust comparison based on class index
    if XRAY_CLASS_INDEX == 0:
        is_xray = score <= (1 - THRESHOLD)
    else:
        is_xray = score >= THRESHOLD
    return is_xray, score
 
def predict(image_bytes, disease: str):
    is_xray, validator_score = validate_xray(image_bytes)
    if not is_xray:
        return {
            'valid': False,
            'error': 'Not a chest X-ray. Please upload a valid image.',
            'validator_score': round(validator_score, 4)
        }
    arr = preprocess(image_bytes)
    model = tb_model if disease == 'tb' else pn_model
    score = float(model.predict(arr, verbose=0)[0][0])
    return {
        'valid': True,
        'disease': disease,
        'prediction': 'positive' if score >= 0.5 else 'negative',
        'confidence': round(score, 4),
        'validator_score': round(validator_score, 4)
    }
