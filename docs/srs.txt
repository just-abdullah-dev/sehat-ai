1. Introduction
1.1 Purpose
The purpose of this document is to define the detailed software requirements for the Sehat AI mobile application. The application leverages Artificial Intelligence (AI) and Machine Learning (ML) to detect Tuberculosis (TB) and Pneumonia from chest X-ray images. The system aims to provide early detection, progress tracking, and an accessible diagnostic tool for patients and healthcare providers, especially in regions with limited access to radiologists.
1.2 Scope
Sehat AI will be a mobile application developed using React Native with Expo, supported by a Python-based backend that integrates trained ML models (EfficientNet-B0 and DenseNet) for image analysis. The backend will handle API requests, user authentication, data storage (PostgreSQL), and report generation.
Key functionalities include:
•	Uploading chest X-ray images.
•	AI-based prediction (TB, Pneumonia, or Normal).
•	Progress tracking with scan history.
•	User medical profile management.
•	PDF report generation for sharing with doctors.
•	Storing user symptoms, medications, and scan improvements.

2. Overall Description
2.1 Product Perspective
Sehat AI will act as a bridge between users (patients or doctors) and the trained ML models for disease detection. It will integrate:
•	A frontend mobile interface (React Native).
•	A Python-based backend server using FastAPI or Flask.
•	ML Models trained on open medical datasets such as TBX11K and ChestX-ray14.
•	Database (PostgreSQL) for storing user data, scan records, and reports.
System Architecture
1.	Mobile App (Frontend):
o	User authentication and registration.
o	Image upload interface (camera/gallery).
o	Display results and history.
o	Generate and view reports.
2.	Backend (Server):
o	RESTful API endpoints.
o	Integration with ML models for inference.
o	Manage user profiles, scans, and data persistence.
3.	Machine Learning Models:
o	EfficientNet-B0 for Pneumonia detection.
o	DenseNet (TBX11K) for Tuberculosis detection.
o	Models will be fine-tuned and optimized for mobile response times.
4.	Database (PostgreSQL):
o	Stores user info, medical history, scan metadata, and reports.

2.4 Constraints
•	Platform: Android & iOS via React Native (Expo).
•	Backend: Python (FastAPI / Flask).
•	Database: PostgreSQL.
•	Image size limit: ≤ 10 MB per upload.
•	Performance constraint: AI inference < 5 seconds.
•	Security: HIPAA-inspired data protection practices.
________________________________________
2.5 Assumptions and Dependencies
•	Users have smartphones with internet access.
•	Uploaded X-rays are in readable format (JPEG/PNG).
•	Model predictions depend on dataset quality and preprocessing.
•	Cloud hosting for backend and storage (e.g., AWS, Render, or Vercel).

3. Specific Requirements
3.1 Functional Requirements
FR1: User Management
•	Users can register using email and password.
•	Authentication tokens (JWT) ensure secure sessions.
•	Users can reset passwords.
FR2: Image Upload
•	The app must allow users to capture or upload a chest X-ray.
•	Validate image format and size.
•	Store metadata in the database.
FR3: Disease Prediction
•	The backend ML service receives the image.
•	Model predicts disease (TB / Pneumonia / Normal).
•	Response returned in JSON format with confidence scores.
FR4: History & Progress Tracking
•	Each scan result is stored with a timestamp.
•	Graphs show disease progression over time.
FR5: Report Generation
•	Generate a PDF report containing:
o	User info.
o	Scan result & confidence.
o	Model used.
o	Progress chart (if applicable).
o	Medical recommendations.
FR6: Notifications
•	Push notifications for scheduled scans or medication.
FR7: Data Management
•	Secure CRUD operations for user data and history.
•	Admin can delete or update records if necessary.
________________________________________
3.2 Non-Functional Requirements
Category	Requirement
Performance	Inference response ≤ 5 seconds, App launch time ≤ 3 seconds.
Security	Encrypted communication (HTTPS, JWT), data anonymization.
