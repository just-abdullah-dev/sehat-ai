Chapter 1: Project Vision

1.1 Problem Statement
Tuberculosis (TB) and Pneumonia are among the leading causes of morbidity and mortality worldwide, especially in developing regions. Early detection plays a critical role in treatment success and survival; however, the shortage of skilled radiologists and healthcare facilities causes delays and inaccuracies in diagnosis.
Traditional diagnosis depends on manual interpretation of chest X-rays, which is often time-consuming, subjective, and prone to human error. Moreover, hospital-based diagnosis can be costly and inaccessible to patients in remote areas.
Currently, there are no mobile-based, AI-powered solutions in Pakistan and similar regions that can assist in real-time, automated detection of TB and Pneumonia from chest X-rays. Hence, there is a strong need for an intelligent, cost-effective, and user-friendly AI mobile application that facilitates early detection and continuous monitoring of these diseases.

1.2 Business Opportunity
Market Need
•	Over 10 million people are diagnosed with TB each year (WHO, 2024).
•	Pneumonia remains one of the leading causes of child mortality worldwide.
•	Developing countries, including Pakistan, face a severe shortage of diagnostic specialists.
Opportunity
The proposed application, Sehat AI, addresses these challenges by leveraging Artificial Intelligence (AI) and mobile technology to automate X-ray image analysis. It offers a low-cost, scalable, and accessible diagnostic platform suitable for both urban and rural populations.
Competitive Advantage
•	Real-time AI-based detection for TB and Pneumonia.
•	Accessibility through mobile devices without the need for specialized hardware.
•	Lower cost compared to hospital diagnostics.
•	Support for multilingual and user-friendly interfaces.
•	Scalable solution for future healthcare integrations.

1.3 Objectives
Primary Objectives
1.	Develop a cross-platform mobile application for TB and Pneumonia detection from chest X-rays.
2.	Train and Integrate machine learning models to achieve reliable and precise results.
3.	Provide a simple and intuitive user interface for image upload and instant predictions.
4.	Maintain a medical history record with progress visualization.
5.	Enable report generation and sharing for professional use by doctors or healthcare institutions.
Secondary Objectives
1.	Ensure data privacy and user security through encryption and authentication.
2.	Support multilingual interfaces (English and Urdu).
3.	Establish the foundation for adding other respiratory disease detections in future versions.

1.4 Project Scope
1.4.1 Included Features
•	AI-based classification of chest X-rays into TB, Pneumonia, or Normal.
•	Image upload from camera or gallery.
•	Real-time disease prediction with confidence percentage.
•	User authentication and secure profile management.
•	Storage of scan history and tracking of disease progression.
•	PDF report generation for medical use.
•	Notifications and reminders for follow-up scans or medication schedules.
•	Responsive and user-friendly UI developed using React Native (Expo).

1.4.2 Excluded Features
•	Integration with hospital EMR systems.
•	Telemedicine or live doctor consultations.
•	Integration with wearable medical devices.
•	Offline model inference (TensorFlow Lite-based).
•	On-device data analysis or deep learning model retraining by users.

1.5 Constraints
Type	Description
Technical	Requires stable internet connection for model inference; image upload limited to ≤10 MB.
Hardware	Compatible with Android (9+) and iOS (13+) devices.
Security	All communications via HTTPS; sensitive data encrypted; JWT-based authentication.
Performance	Prediction latency ≤10 seconds for optimal user experience.
Resource	Limited dataset availability and computing resources for model training.

1.6 Feasibility
1.6.1 Technical Feasibility
The project uses React Native with Expo for cross-platform mobile development and a Python-based backend (FastAPI) that integrates deep learning models built using TensorFlow/Keras.
Both frameworks are open-source, reliable, and well-supported, ensuring smooth development and deployment. Cloud-based deployment (AWS/Render/Railway) will support scalability and high availability.
1.6.2 Resource Feasibility
The project requires:
•	Developer laptops with Python and Node.js environments.
•	Access to GPUs for training and fine-tuning ML models. (Google Colab).
•	Publicly available datasets such as TBX11K and Chest X-Ray Images (Pneumonia).
•	Hosting services for backend and database (PostgreSQL).

1.6.3 Time Feasibility
The project development is divided into structured phases:
1.	Phase 1: Requirement analysis and dataset preparation.
2.	Phase 2: ML model training, evaluation, and optimization.
3.	Phase 3: Frontend and backend integration with API development.
4.	Phase 4: Testing, performance tuning, and deployment.
5.	Phase 5: Final review, documentation, and presentation.

1.6.4 Market Feasibility
Sehat AI fulfills a significant healthcare gap in low-resource areas where diagnostic services are limited. Its mobile-first design provides accessibility and affordability, making it viable for real-world use.
With increasing smartphone adoption, this solution holds high potential for public health impact, especially in developing regions.

Chapter 2: Software Requirements Specification

2.1 List of Features
The Sehat AI mobile application provides the following key features:
•	X-ray Upload:
Provides functionality for uploading chest X-ray images directly through the device camera or photo gallery for analysis.
•	AI Detection:
Train and Integrates Machine Learning models using TBX11K for TB and NIH ChestX-ray14 dataset predict whether the chest X-ray indicates Tuberculosis, Pneumonia, or Normal condition.
•	Prediction Confidence Score:
Displays a confidence percentage representing how certain the model is about its prediction, allowing users and doctors to assess reliability.
•	Multilingual Interface:
Provides English and Urdu language support to make the application more accessible to local users in Pakistan and similar regions.
•	Dark Mode Support:
Offers a light and dark mode toggle to enhance usability and comfort under different lighting conditions.
•	Secure Data Handling:
Ensures that all communications are encrypted (HTTPS) and sensitive files are stored securely, protecting patient data and confidentiality.
•	User Authentication:
Enables secure user registration, login, and logout using JWT-based authentication, ensuring privacy and protection of personal data.
•	Profile Management:
Allows users to edit and update personal information, record symptoms, and log medications for better tracking of health conditions.
•	Scan History:
Stores and displays all previous scans and results in chronological order for easy access and comparison over time.
•	Report Generation:
Enables users to generate, download, and share PDF reports containing scan details, predictions, and confidence scores for professional medical review.

2.2 Functional Requirements
FR1: X-ray Upload
•	The app shall allow users to upload chest X-ray images from the gallery or camera.
•	The app shall validate image format (JPEG/PNG) and size (≤ 10 MB).
•	The app shall preview uploaded images before submission.
FR2: AI Model Prediction
•	The backend shall classify the image as TB, Pneumonia, or Normal.
•	The backend shall return a confidence score (0–100%).
•	The response time for prediction shall be ≤ 10 seconds.
FR3: User Authentication
•	The system shall allow users to register with name, email, and password.
•	The system shall allow users to log in and log out securely using JWT tokens.
•	The system shall allow users to reset password via registered email.
FR4: Profile Management
•	The user shall be able to update profile information, including symptoms and medications.
•	The system shall store and retrieve user medical history securely.
FR5: Multilingual Support
•	The app shall allow the user to switch between English and Urdu.
FR6: Security
•	The system shall use HTTPS for secure communication.
•	All user credentials shall be encrypted.
FR7: Scan History and Tracking
•	The system shall store all user scans with date and time.
•	The system shall display results chronologically.
•	The system shall visualize disease progression through graphs.
FR8: Report Generation
•	The system shall generate PDF reports including patient details, results, and confidence scores.
•	The report shall be downloadable and shareable via email or messaging apps.

Chapter 3 — System Overview

This chapter explains how the whole system of our FYP “Sehat AI” works from backend, frontend, database, and machine learning perspectives. Since our platform detects TB and Pneumonia from chest X-rays, we have a FastAPI backend that loads two trained .h5 models, a React Native frontend app, and a PostgreSQL database to store users and scan history.
________________________________________
3.1 Architectural Design
3.1.1 High Level Overview
Our system is divided into three main layers:
1.	Frontend (React Native App)
o	Allows users to register, login, upload X-ray images, and view predictions.
o	Communicates with backend through REST APIs.
2.	Backend (FastAPI)
o	Loads two ML models (tb_model.h5 and pneumonia_model.h5) using TensorFlow/Keras.
o	Provides image upload API, prediction API, authentication API, and history API.
o	Also stores user data and prediction results in PostgreSQL.
3.	Database (PostgreSQL)
o	Stores user accounts
o	Stores scan history including predictions, confidence scores, model used, timestamps, and file URLs
High-Level Architecture Diagram 
 
________________________________________
3.1.2 Workflow Overview
Below is the step-by-step process followed when a user uploads an X-ray:
1.	User uploads a chest X-ray.
2.	Backend receives the image and validates it.
3.	Depending on user selection (TB or Pneumonia), appropriate model is used.
4.	TensorFlow loads .h5 model and performs inference.
5.	User logs in from the mobile app.
6.	Backend returns:
o	Result (Normal or Positive)
o	Confidence score
o	Processing time
7.	Backend saves scan info in the database.
8.	User can view the result or history.
