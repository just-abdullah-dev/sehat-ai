1. Introduction
1.1 Purpose
The purpose of this document is to specify all frontend requirements, functionalities, and user interface behaviors for the Sehat AI mobile application developed using React Native with Expo.
The app allows users to:
•	Upload chest X-rays.
•	Get instant AI-based diagnostic predictions (TB, Pneumonia, or Normal).
•	Track medical history and progress visually.
•	View and download reports.
•	Record symptoms and medications.
This SRS will serve as a reference for frontend developers to design, implement, and test the UI/UX according to functional and non-functional requirements.
________________________________________
1.2 Scope
The Sehat AI Mobile App provides an accessible platform for healthcare users to detect TB and Pneumonia using ML-powered backend services.
The mobile app will:
•	Run on Android and iOS using Expo (React Native).
•	Communicate securely with a Python FastAPI/Flask backend.
•	Support push notifications, image uploads, and PDF report downloads.
Main Modules:
1.	User Authentication (Login/Signup)
2.	Dashboard (Upload & View Predictions)
3.	Scan History (Progress Tracking)
4.	Profile & Medical Records
5.	Report Generation & Sharing
6.	Notifications & Reminders

2. Overall Description
2.1 Product Perspective
The frontend is a standalone React Native Expo app acting as a client to the backend ML and database APIs.
The app will feature:
•	A clean, responsive interface.
•	Seamless API integration for real-time predictions.
•	Local storage for session management (using AsyncStorage).

3. Specific Requirements
3.1 Functional Requirements
FR1: User Authentication
•	Signup Page:
o	Input: Name, Email, Password, Confirm Password.
o	Validation for strong password.
o	API: /auth/signup
•	Login Page:
o	Input: Email, Password.
o	API: /auth/login
o	Store JWT in AsyncStorage.
•	Logout:
o	Clear local storage, redirect to login screen.
FR2: Dashboard / Home Screen
•	Display greeting (e.g., “Welcome, Abdullah”).
•	Buttons for:
o	Upload X-ray → opens image picker or camera.
o	View Last Scan → fetch from /user/history.
•	API Integration:
o	POST /predict/ → upload image, receive result.
FR3: Image Upload & Prediction
•	Use Expo ImagePicker for upload.
•	Display image preview.
•	On “Analyze” button:
o	Show loading indicator while fetching results.
o	Receive { result, confidence } JSON.
o	Display:
o	Result: Pneumonia Detected
o	Confidence: 92.4%
•	Store result locally and in backend history.
FR4: History Screen
•	Fetch data via /user/history.
•	Display list:
•	[Date] - [Result] - [Confidence]
•	Graph of progress (using react-native-chart-kit).
•	Search filter by date or result type.
FR5: Reports
•	Button “Generate Report” triggers /user/report/:id.
•	PDF downloaded or previewed in-app using expo-file-viewer.
•	Option to “Share Report” (WhatsApp, Email).
FR6: Profile Screen
•	Fields:
o	Name, Age, Gender, Phone, Email.
o	Symptoms input (textarea).
o	Medicines input (list).
•	API endpoints:
o	GET /user/profile
o	PUT /user/update
•	Profile image upload (optional).
FR7: Notifications
•	Expo Push Notifications for:
o	Reminder to scan after 14 days.
o	Medication schedule (if enabled).
•	API: /user/notifications
FR8: Settings
•	Language toggle (English/Urdu).
•	Dark/Light theme.
•	Logout & Delete Account options.
________________________________________
3.2 Non-Functional Requirements
Category	Requirement
Performance	App should load within 3 seconds; API response < 5 seconds.
Security	Use HTTPS, JWT authentication, and local encryption (AsyncStorage).
Reliability	Offline caching of last 10 scans.
Usability	Minimal navigation depth (max 3 clicks to main actions).
Portability	Works identically on Android and iOS.
Maintainability	Modular code structure with reusable components.
Accessibility	Large fonts, contrast themes, and voice-over support.
